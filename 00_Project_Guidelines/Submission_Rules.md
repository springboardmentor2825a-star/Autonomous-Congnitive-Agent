# Submission Rules

1. Fork this repository.
2. Do not push directly to the mentor repository.
3. Create a folder: Intern_Submissions/Your_Full_Name/
4. All code and documents must stay inside your folder.
5. Your project must include a README.md.
6. Commits should be meaningful and regular.
7. Push your work to your fork.
8. Create a Pull Request for mentor review.
9. Only reviewed Pull Requests will be merged.

Breaking these rules may disqualify submissions.
